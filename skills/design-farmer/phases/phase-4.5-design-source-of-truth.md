# Phase 4.5: Design Source of Truth (DESIGN.md)

Generate a `DESIGN.md` file at `{systemPath}/DESIGN.md` so the design artifact always lives beside the generated design-system work it describes.

## 4.5.3 Subsequent Run Behavior

If DESIGN.md already exists:
1. Read the existing file
2. Append new entries to the Decisions Log
3. Update sections that have changed (color, typography, spacing)
4. Preserve the history — never delete past decisions
5. Add a `## Revision History` section if not present:

```
## Revision History
| Date | Action | Sections Changed |
|------|--------|-----------------|
| {date} | Initial generation | All |
| {date} | Updated palette | Color System, Decisions Log |
```

## 4.5.4 Phase 8 Integration

Update Phase 8 reviewer prompts to include:

```
Calibrate your review against the documented design decisions.
Flag any implementation that deviates from the documented direction as HIGH severity."
```
